Introduction to Semantic HTML

Semantic HTML introduces elements that clearly describe their meaning, making web pages easier to read and better optimized for search engines and assistive technologies.

Benefits of Using Semantic Elements

  • Improves SEO and search engine understanding of content
  • Enhances accessibility for screen readers
  • Makes code more readable and maintainable

Examples of Semantic Elements

<article>

Used for self-contained content like blog posts or news stories. Can be distributed independently from the rest of the site.

<section>

Represents a standalone section of related content, usually with a heading. Ideal for grouping content logically.

<nav>

Used to define navigation links. Helps search engines and screen readers identify site navigation.

<header> and <footer>

<header> introduces the page or a section, while <footer> wraps up with related links or copyright.
